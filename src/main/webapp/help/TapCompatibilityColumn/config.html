This column provides quick information on test result compatibility with another job.
In order to be able to leverage this feature, you need to stick with a simple name convention, where corresponding jobs follow certain naming pattern.
You could e.g. have the following jobs:
job namedescription
FeatureA-nightlytest for feature A running on your nightly builds
FeatureB-nightlytest for feature B running on your nightly builds
FeatureA-goldenbuild that generates golden (desired, probably made up) result TAP file, for feature A
FeatureB-goldendtto (golden file generator) for feature B
